C · 砍掉 chat,提交页只管提交

quick.html 只做一件事:展示 card + 收集 attest。想问 AI 问题的用户去另一个入口 /employee/ask.html(一个独立 chat 页)。

代价:员工常见需求("这张该算哪个项目")就得跳页,违背 agent 流"一站搞定"的承诺
我的倾向:B(内联问答行)
理由:
• 你的 attest 原则要求 card 是"glance+tap",chat drawer 会破坏这个,A 藏起来等于没有
• 但 chat 是 agent 流的核心能力,砍了 C 就退化成"有 AI 的 form"
• 内联一行最小打扰:不问的人看不见(只是一行占位),问的人点一下就展开

功能约束:这一行的 chat 只接受和当前这张票相关的问题。跨越当前 receipt 的问题(比如"我这个月总共花了多少")继续走首页的全局 agent 入口。这样 card 的 chat context 就不会污染,回答可以更精准。
